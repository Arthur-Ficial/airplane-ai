What does @MainActor do in Swift?
@MainActor is a global actor that ensures code runs on the main thread. The compiler enforces this at build time — you get errors if you call main-actor code from another context without await.

Use it for any UI code, view models, or state shared with SwiftUI views. Keep heavy work (network, disk, compute) off it.

Explain closures in one paragraph.

SQLite vs Core Data for a Mac app — which should I pick?
Depends what you value:
  • SQLite (raw) — full control, portable .sqlite, you write migration SQL.
  • SwiftData / Core Data — SwiftUI bindings, automatic change tracking, built-in migrations.
For a SwiftUI Mac app: SwiftData is the pragmatic default.

Debugging
Why would a SwiftUI view not update when my @Observable model changes?
Most common causes:
  1. Mutating the model off the MainActor.
  2. The view holds a copy instead of observing the original.
  3. You're using let on a value that changes.
Mark your model @MainActor and inject via @Bindable or @Environment.
